# 后端接口参考文档（前端开发专用）

本文档定义了一套校园二手交易平台的核心 API。前端需按此规范对接**用户服务、商品服务、交易订单服务**，所有请求统一通过**网关服务**转发。

## 基础信息

- **网关统一地址**：`/api`（开发环境可配置为 `http://localhost:8080/api`）
- **认证方式**：除登录接口外，其余请求需在 Header 中携带 `Authorization: Bearer {token}`（登录成功后获得）
- **数据格式**：请求与响应均为 `application/json`；字符编码 UTF-8
- **统一响应结构**：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": { ... }
  }
  1. 用户服务
核心数据模型 User
字段名	类型	说明
userId	string	用户唯一 ID（学号）
studentId	string	学号
name	string	真实姓名
phone	string	手机号
college	string	学院
verified	boolean	认证状态（true=已认证）
avatar	string	头像 URL
nickname	string	昵称
contact	string	联系方式（微信/QQ）
API 列表
1.1 用户登录（无需认证）
接口：POST /api/user/login

请求体：

json
{
"studentId": "20240001",
"password": "123456"
}
响应：

json
{
"code": 200,
"data": {
"token": "eyJhbGciOiJIUzI1NiIs...",
"userId": "20240001",
"nickname": "张三"
}
}
1.2 获取用户信息
接口：GET /api/user/{userId}

路径参数：userId 用户ID

响应：返回 User 对象

1.3 修改用户信息
接口：PUT /api/user/info

请求体：需修改的字段（如 nickname, avatar, contact）

响应：更新后的 User 对象

2. 商品服务
   核心数据模型 Product
   字段名	类型	说明
   productId	string	商品唯一 ID
   category	string	分类（例如：数码、书籍、生活用品）
   title	string	标题
   description	string	描述
   price	number	价格（单位：元）
   condition	string	成色（全新/几乎全新/有使用痕迹）
   imageUrls	array	图片 URL 列表
   publishTime	string	发布时间（ISO 8601）
   status	string	商品状态：ON_SALE / OFF_SALE / REVIEW
   API 列表
   2.1 获取商品详情
   接口：GET /api/product/{productId}

路径参数：productId

响应：Product 对象

2.2 查询商品列表
接口：GET /api/product/list

查询参数（均为可选）：

category 分类

keyword 标题/描述关键字

status 状态（默认 ON_SALE）

page 页码，默认 1

size 每页数量，默认 10

响应：

json
{
"total": 35,
"list": [ Product1, Product2, ... ]
}
2.3 发布商品
接口：POST /api/product

请求体：Product 对象（不含 productId, publishTime, status）

响应：新创建的 Product（含服务端生成的字段）

2.4 修改商品
接口：PUT /api/product

请求体：完整的 Product 对象

响应：更新后的 Product

2.5 下架/删除商品
接口：DELETE /api/product/{productId}

路径参数：productId

响应：空或成功消息

3. 交易订单服务
   核心数据模型 Order
   字段名	类型	说明
   orderId	string	订单唯一 ID
   userId	string	买家用户 ID
   productId	string	商品 ID
   price	number	交易价格（下单时锁定商品价格）
   tradeMethod	string	交易方式：FACE_TO_FACE / SELF_PICKUP
   status	string	订单状态：PENDING / PAID / COMPLETED / CANCELLED
   createTime	string	创建时间
   finishTime	string	完成时间（仅当 status=COMPLETED 时有值）
   API 列表
   3.1 创建订单
   接口：POST /api/order

请求体：

json
{
"productId": "PROD_001",
"tradeMethod": "FACE_TO_FACE"
}
响应：新创建的 Order 对象

3.2 查询订单详情
接口：GET /api/order/{orderId}

路径参数：orderId

响应：Order 对象（应联查出商品标题、图片等展示信息）

3.3 查询我的订单列表
接口：GET /api/order/list

查询参数：

status 可选，过滤订单状态

page / size

响应：分页的订单列表（建议后端返回订单 + 商品快照信息）

3.4 取消订单
接口：PUT /api/order/cancel

请求体：

json
{ "orderId": "ORD_123" }
响应：更新后的订单（status = CANCELLED）

4. 网关路由映射
   前端所有请求均指向网关，由网关转发至对应服务：

路由前缀	目标服务
/api/user/**	用户服务
/api/product/**	商品服务
/api/order/**	订单服务
示例：

登录 → /api/user/login

商品列表 → /api/product/list

创建订单 → /api/order

前端开发对接规范
1. 全局配置
   javascript
   // config.js
   export const API_BASE = '/api';          // 开发时代理到网关
   export const USE_MOCK = false;           // 对接真实接口时关闭 mock
   export const TIMEOUT = 10000;
2. 封装请求函数（示例）
   javascript
   async function request(endpoint, options = {}) {
   const url = `${API_BASE}${endpoint}`;
   const headers = {
   'Content-Type': 'application/json',
   ...(localStorage.getItem('token') && { Authorization: `Bearer ${localStorage.getItem('token')}` })
   };
   const response = await fetch(url, { ...options, headers });
   const json = await response.json();
   if (json.code !== 200) throw new Error(json.message);
   return json.data;
   }
3. 错误处理建议
   401：清除本地 token，跳转登录页。

404：提示“资源不存在”。

网络错误：显示“网络异常，稍后重试”，并提供重试按钮。

4. 字段映射注意事项
   图片 URL 可能为相对路径，前端需拼接完整地址（由后端配置 imageBaseURL）。

价格以“元”为单位，前端展示保留两位小数。

时间字段统一使用 new Date(isoString).toLocaleString() 格式化。

5. 联调 Mock 数据策略
   在真实接口未就绪时，前端可使用相同的字段结构编写本地 mock，确保 UI 与真实数据格式兼容。建议将 mock 数据单独放在 mock/ 目录，并通过环境变量切换。

附录：错误码说明
code	含义
200	成功
400	请求参数错误
401	未登录或 token 失效
403	无权限（如删除他人商品）
404	资源不存在
409	业务冲突（如商品已下架）
500	服务器内部错误
