# 订单服务使用说明

## 功能概述

订单服务提供以下核心功能：

### 1. 订单管理
- ✅ 创建订单
- ✅ 取消订单
- ✅ 订单状态跟踪（待支付 / 待交付 / 已完成 / 已关闭）

### 2. 交易方式
- ✅ 面交（FACE_TO_FACE）
- ✅ 自提（SELF_PICKUP）
- ✅ 交易地点记录
- ✅ 交易时间记录

### 3. 个人中心
- ✅ 我的订单查询
- ✅ 按状态筛选订单

---

## 数据库初始化

执行 `src/main/resources/order_db.sql` 文件创建订单表：

```sql
CREATE TABLE IF NOT EXISTS orders (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '订单ID',
    order_no VARCHAR(64) NOT NULL UNIQUE COMMENT '订单编号',
    user_id INT NOT NULL COMMENT '用户ID',
    product_id INT NOT NULL COMMENT '商品ID',
    product_name VARCHAR(255) NOT NULL COMMENT '商品名称',
    product_image VARCHAR(500) COMMENT '商品图片',
    price DECIMAL(10, 2) NOT NULL COMMENT '单价',
    quantity INT NOT NULL DEFAULT 1 COMMENT '数量',
    total_amount DECIMAL(10, 2) NOT NULL COMMENT '总金额',
    status VARCHAR(20) NOT NULL DEFAULT 'PENDING_PAYMENT' COMMENT '订单状态: PENDING_PAYMENT-待支付, PENDING_DELIVERY-待交付, COMPLETED-已完成, CLOSED-已关闭',
    trade_type VARCHAR(20) NOT NULL COMMENT '交易方式: FACE_TO_FACE-面交, SELF_PICKUP-自提',
    trade_location VARCHAR(255) COMMENT '交易地点',
    trade_time DATETIME COMMENT '交易时间',
    remark VARCHAR(500) COMMENT '备注',
    cancel_reason VARCHAR(500) COMMENT '取消原因',
    cancel_time DATETIME COMMENT '取消时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_user_id (user_id),
    INDEX idx_order_no (order_no),
    INDEX idx_status (status),
    INDEX idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

---

## 配置说明

在 `bootstrap.yml` 中配置数据库连接：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/order_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: root
    password: root
```

## API接口列表

### 1. 创建订单
**接口**: `POST /orders`

**请求参数**:
- userId: 用户ID（query参数）
- Request Body:
```json
{
  "productId": 1,
  "productName": "二手自行车",
  "productImage": "http://example.com/image.jpg",
  "price": 299.00,
  "quantity": 1,
  "tradeType": "FACE_TO_FACE",
  "tradeLocation": "学校东门",
  "remark": "希望周末交易"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "id": 1,
    "orderNo": "ORD202606081200001234",
    "userId": 1001,
    "status": "PENDING_PAYMENT",
    "statusDesc": "待支付",
    "tradeType": "FACE_TO_FACE",
    "tradeTypeDesc": "面交"
  }
}
```

---

### 2. 取消订单
**接口**: `POST /orders/cancel`

**请求参数**:
- userId: 用户ID（query参数）
- Request Body:
```json
{
  "orderId": 1,
  "cancelReason": "不想买了"
}
```

---

### 3. 更新订单状态
**接口**: `PUT /orders/status`

**请求参数**:
- orderId: 订单ID（query参数）
- status: 订单状态（query参数）
  - PENDING_PAYMENT: 待支付
  - PENDING_DELIVERY: 待交付
  - COMPLETED: 已完成
  - CLOSED: 已关闭
- tradeLocation: 交易地点（可选，query参数）

**示例**: `PUT /orders/status?orderId=1&status=PENDING_DELIVERY&tradeLocation=学校东门`

---

### 4. 查询订单详情
**接口**: `GET /orders/{orderId}`

**示例**: `GET /orders/1`

---

### 5. 订单列表查询
**接口**: `GET /orders/list`

**请求参数**:
- userId: 用户ID（可选）
- status: 订单状态（可选）
- keyword: 关键词搜索（可选，支持商品名称和订单号）
- pageNum: 页码，默认1
- pageSize: 每页数量，默认10

**示例**: `GET /orders/list?userId=1001&status=PENDING_PAYMENT&pageNum=1&pageSize=10`

---

### 6. 我的订单（个人中心）
**接口**: `GET /orders/my`

**请求参数**:
- userId: 用户ID（必填）
- status: 订单状态（可选）
- pageNum: 页码，默认1
- pageSize: 每页数量，默认10

**示例**: 
- 查询所有订单: `GET /orders/my?userId=1001`
- 查询待支付订单: `GET /orders/my?userId=1001&status=PENDING_PAYMENT`
- 查询已完成订单: `GET /orders/my?userId=1001&status=COMPLETED`

---

## 订单状态说明

| 状态码 | 中文描述 | 说明 |
|--------|---------|------|
| PENDING_PAYMENT | 待支付 | 订单已创建，等待支付 |
| PENDING_DELIVERY | 待交付 | 已支付，等待交付 |
| COMPLETED | 已完成 | 交易完成 |
| CLOSED | 已关闭 | 订单已取消或关闭 |

| 方式代码 | 中文描述 | 说明 |
|---------|---------|------|
| FACE_TO_FACE | 面交 | 双方面对面交易 |
| SELF_PICKUP | 自提 | 买家自行提取商品 |

## 启动服务

1. 确保MySQL数据库已启动
2. 执行SQL脚本创建数据库表
3. 配置 `bootstrap.yml` 中的数据库连接信息
4. 启动Nacos服务注册中心
5. 运行 `OrderServiceApplication.java`

访问Swagger文档: http://localhost:8083/doc.html

---

## 注意事项

1. **权限控制**: 只能操作自己的订单（通过userId验证）
2. **订单取消限制**: 已完成(COMPLETED)和已关闭(CLOSED)的订单不能再次取消
3. **订单号生成**: 自动生成唯一订单号，格式为 ORD + 时间戳 + 4位随机数
4. **事务管理**: 创建订单、取消订单、更新状态都使用事务保证数据一致性
5. **分页查询**: 列表查询支持分页，默认每页10条记录
