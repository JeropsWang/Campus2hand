## 预防措施

### 1. IntelliJ IDEA 编码设置

#### 全局编码设置
1. **File → Settings → Editor → File Encodings**
2. 设置以下选项：
   - **Global Encoding**: `UTF-8`
   - **Project Encoding**: `UTF-8`
   - **Default encoding for properties files**: `UTF-8`
   - ✅ 勾选 `Transparent native-to-ascii conversion`

#### 项目编码配置
在项目根目录创建 `.idea/encodings.xml`：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<project version="4">
  <component name="Encoding">
    <file url="file://$PROJECT_DIR$/order-service/src/main/java/**/*.java" charset="UTF-8" />
    <file url="file://$PROJECT_DIR$/order-service/src/main/resources/**/*.yml" charset="UTF-8" />
    <file url="file://$PROJECT_DIR$/order-service/src/main/resources/**/*.properties" charset="UTF-8" />
  </component>
</project>
```

### 2. Maven 编码配置

在 `pom.xml` 中确保有如下配置（父pom已配置）：
```xml
<properties>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
</properties>
```

### 3. Git 编码配置

创建或更新 `.gitattributes` 文件：
```
*.java text eol=lf charset=utf-8
*.yml text eol=lf charset=utf-8
*.properties text eol=lf charset=utf-8
*.xml text eol=lf charset=utf-8
*.md text eol=lf charset=utf-8
```

## 安全编辑建议

### ❌ 避免的操作
```powershell
# 不要使用PowerShell直接替换含中文的文件
(Get-Content file.java -Raw) -replace 'old', 'new' | Set-Content file.java
```

### ✅ 推荐的操作

#### 方法1：使用IDE的查找替换功能
- **Ctrl + R** (Windows/Linux)
- **Cmd + R** (Mac)
- 选择 "In Project" 或 "In Module"

#### 方法2：使用search_replace工具
通过AI助手使用search_replace工具进行精确替换

#### 方法3：如果必须使用命令行
```powershell
# 指定UTF-8编码
Get-Content file.java -Encoding UTF8 -Raw | 
    ForEach-Object { $_ -replace 'javax.validation', 'jakarta.validation' } | 
    Set-Content file.java -Encoding UTF8 -NoNewline
```

## 检查文件编码

### PowerShell 检查
```powershell
# 查看文件编码
(Get-Item filename.java).FullName
[System.IO.File]::ReadAllText("filename.java", [System.Text.Encoding]::UTF8)
```

### IDE 检查
- IDEA右下角状态栏会显示当前文件的编码
- 如果显示GBK或其他编码，点击后可以选择 "Convert to UTF-8"

## 乱码症状识别

### 典型乱码表现
```java
// 错误示例（UTF-8被当作GBK读取）
/**
 * 璁㈠崟鎺у埗鍣?  <- 应该是 "订单控制器"
 */
@Operation(summary = "鍒涘缓璁㈠崟")  <- 应该是 "创建订单"
```

### 正确显示
```java
// 正确示例
/**
 * 订单控制器
 */
@Operation(summary = "创建订单")
```

## 常见问题

### Q1: 为什么会出现乱码？
A: 不同工具对文件编码的处理方式不同，特别是Windows系统默认使用GBK编码。

### Q2: 如何避免乱码？
A: 
- 统一使用UTF-8编码
- 使用IDE进行文件编辑
- 避免使用命令行直接修改含中文的文件

### Q3: Git提交后乱码怎么办？
A: 
- 检查本地文件编码是否正确
- 确保.gitattributes配置正确
- 重新克隆仓库时设置正确的core.autocrlf

### Q4: IDEA提示编码不匹配？
A: 
- 选择 "Convert" 而不是 "Reload"
- Convert会保持显示内容不变，转换文件编码
- Reload会重新读取文件，可能导致乱码
